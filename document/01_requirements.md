# ファイブリーグ風クイズゲーム 要件定義書

## 1. プロジェクト概要

### 1.1 プロダクト名

ファイブクイズ

### 1.2 コンセプト

テレビ番組「ネプリーグ」の「ファイブリーグ」を模した 5 人協力型クイズゲーム。
5 人のプレイヤーがそれぞれ 1 文字ずつ担当し、チームで 5 文字の回答を完成させる。

### 1.3 ターゲットユーザー

- 友人同士でオンラインでクイズを楽しみたい人
- パーティーゲームとして遊びたいグループ

## 3. 機能要件

### 3.1 ユーザー管理

| ID   | 機能             | 詳細                                           |
| ---- | ---------------- | ---------------------------------------------- |
| U-01 | ログイン不要     | ユーザー認証なしで利用可能                     |
| U-02 | ニックネーム入力 | ルーム入室前に任意のニックネームを入力（必須） |
| U-03 | セッション管理   | ブラウザセッションでユーザーを識別             |

### 3.2 ルーム管理

| ID   | 機能         | 詳細                                         |
| ---- | ------------ | -------------------------------------------- |
| R-01 | ルーム作成   | 誰でもルームを作成可能。作成者がホストになる |
| R-02 | ルーム参加   | 共有リンク（URL）からルームに参加            |
| R-03 | 参加人数制限 | 最大 5 人まで参加可能                        |
| R-04 | 5 人必須     | ゲーム開始には 5 人揃う必要がある            |
| R-05 | 途中参加不可 | ゲーム開始後は新規参加不可                   |
| R-06 | ルーム終了   | ホストが退出するとルーム終了                 |

### 3.3 ホスト権限

| ID   | 機能             | 詳細                                            |
| ---- | ---------------- | ----------------------------------------------- |
| H-01 | 回答順指定       | 5 人のプレイヤーの担当文字（1〜5 文字目）を指定 |
| H-02 | ゲーム開始       | 5 人揃った状態でゲームを開始                    |
| H-03 | 次の問題へ       | 結果表示後、次の問題に進む                      |
| H-04 | ゲーム終了       | 任意のタイミングでゲームを終了                  |
| H-05 | プレイヤーキック | ゲーム開始前に特定プレイヤーを退出させる        |

### 3.4 クイズ出題

| ID   | 機能         | 詳細                                           |
| ---- | ------------ | ---------------------------------------------- |
| Q-01 | クイズデータ | 事前に DB に登録された固定クイズセットから出題 |
| Q-02 | 5 文字固定   | 回答は必ず 5 文字                              |
| Q-03 | ランダム出題 | クイズはランダム順で出題                       |
| Q-04 | 問題数無制限 | 問題数の上限なし（ホストが終了するまで継続）   |
| Q-05 | 重複防止     | 同一ゲーム内で同じ問題は出題しない             |

### 3.5 回答入力

| ID   | 機能           | 詳細                                           |
| ---- | -------------- | ---------------------------------------------- |
| A-01 | 手書き入力     | キーボードではなく、タッチ/マウスで手書き      |
| A-02 | 文字種制限なし | ひらがな・カタカナ・漢字・英数字すべて対応     |
| A-03 | 画像保存       | 手書きは文字認識せず、画像（Canvas）として保存 |
| A-04 | 書き直し可能   | 提出前なら何度でも書き直し可能                 |
| A-05 | 回答提出       | 「提出」ボタンで回答を確定                     |
| A-06 | 制限時間なし   | 全員が提出するまで待機                         |

### 3.6 結果表示・判定

| ID   | 機能               | 詳細                                 |
| ---- | ------------------ | ------------------------------------ |
| J-01 | 全員の回答表示     | 全員の手書き回答を一覧表示           |
| J-02 | 正解表示           | 問題文と正解を表示                   |
| J-03 | 自己判定           | 各プレイヤーが自分の文字の正誤を判定 |
| J-04 | 判定ボタン         | 正解 → 赤ボタン、不正解 → 青ボタン   |
| J-05 | チーム正解カウント | 5 文字全員正解の場合にチーム正解数+1 |
| J-06 | 正解数表示         | ゲーム中、チームの正解数を表示       |

### 3.7 UI/UX

| ID    | 機能                 | 詳細                                 |
| ----- | -------------------- | ------------------------------------ |
| UI-01 | ネプリーグ風デザイン | テレビ番組を意識した派手めのデザイン |
| UI-02 | 効果音               | 正解音、不正解音、BGM など           |
| UI-03 | アニメーション       | 最低限の画面遷移アニメーション       |
| UI-04 | レスポンシブ対応     | スマートフォン・PC 両対応            |

## 4. 非機能要件

### 4.1 パフォーマンス

| ID   | 要件             | 詳細                                             |
| ---- | ---------------- | ------------------------------------------------ |
| P-01 | リアルタイム同期 | 500ms 以内にプレイヤー間で状態同期               |
| P-02 | 同時接続         | 1 ルームあたり最大 6 接続（5 プレイヤー + 予備） |

### 4.2 セキュリティ

| ID   | 要件           | 詳細                                |
| ---- | -------------- | ----------------------------------- |
| S-01 | ルーム ID      | 推測困難なランダム文字列（UUID 等） |
| S-02 | 入力サニタイズ | ニックネーム等の XSS 対策           |

### 4.3 ブラウザ対応

| ID   | 要件         | 詳細                             |
| ---- | ------------ | -------------------------------- |
| B-01 | 対応ブラウザ | Chrome, Safari, Edge（最新版）   |
| B-02 | Canvas 対応  | 手書き入力のため Canvas API 必須 |

## 6. ゲームフロー

```
1. ホストがルーム作成
   ↓
2. 共有リンクを他プレイヤーに共有
   ↓
3. 各プレイヤーがニックネームを入力して入室
   ↓
4. 5人揃ったらホストが順番を指定
   ↓
5. ホストがゲーム開始
   ↓
6. 問題が表示される
   ↓
7. 各プレイヤーが担当文字を手書きで入力→提出
   ↓
8. 全員提出後、結果表示
   ↓
9. 各自が正解/不正解を判定
   ↓
10. ホストが「次の問題へ」または「ゲーム終了」
    ↓
   (6に戻る or 終了画面へ)
```

## 7. 用語定義

| 用語       | 定義                                                   |
| ---------- | ------------------------------------------------------ |
| ホスト     | ルームを作成したプレイヤー。ゲーム進行の権限を持つ     |
| ルーム     | 5 人が集まってゲームを行う仮想空間                     |
| 担当文字   | 各プレイヤーが回答する 1 文字（1〜5 文字目のいずれか） |
| チーム正解 | 5 人全員の文字が正解だった場合にカウントされる         |
